UNDERSTANDING RB BASIC COMPILATION:
-----------------------------------

RB BASIC uses a TWO-STEP compilation process:

Step 1: BASIC → C Transpilation
  - rbbasic.exe reads your .bas file
  - Generates output.c (C source code)
  - Generates compile.bat (compilation script)
  - Both created in same directory as .bas file

Step 2: C → EXE Compilation
  - compile.bat runs GCC to compile output.c
  - Creates program.exe (hardcoded name)
  - Uses MinGW32 GCC from rb.config

WORKFLOW DIAGRAM:
-----------------

  yourfile.bas
       |
       v
  [rbbasic.exe] -----> output.c
       |                   |
       +------------> compile.bat
                           |
                           v
                      [GCC Compile]
                           |
                           v
                      program.exe  (ALWAYS this name)


WHAT THE CORRECTED IDE DOES:
-----------------------------

1. RunCompile() Function:
   
   a) Saves your .bas file
   
   b) Runs: ..\rbbasic.exe "yourfile.bas"
      - Working directory: same as your .bas file
      - This generates output.c and compile.bat
      - Captures and shows transpiler output
   
   c) Checks if compile.bat was created
      - If not found, reports error
   
   d) Runs: cmd.exe /c compile.bat
      - Working directory: same as your .bas file  
      - Compiles output.c to program.exe
      - Captures and shows GCC output
   
   e) Checks if program.exe was created
      - Shows success or failure message

2. RunProgram() Function:
   
   a) Looks for program.exe in source directory
      - Example: If source is C:\test\hello.bas
      - Looks for: C:\test\program.exe
   
   b) Runs program.exe with ShellExecute
      - Opens in separate window
      - Working directory set correctly

3. RunCompileAndRun() Function:
   
   a) Calls RunCompile()
   b) Checks output for "Compilation SUCCESSFUL"
   c) If successful, calls RunProgram()
   d) Brief 500ms pause so user can see messages

    ├── output.c            ← Generated by rbbasic.exe
    ├── compile.bat         ← Generated by rbbasic.exe
    └── program.exe         ← Generated by compile.bat

COMPILE.BAT EXPLAINED:
-----------------------

The rbbasic.exe transpiler generates compile.bat which contains
something like:

  @echo off
  gcc output.c -o program.exe -I..\sdl2\include -L..\sdl2\lib -lmingw32 -lSDL2main -lSDL2
  if errorlevel 1 (
    echo Compilation failed!
    pause
  ) else (
    echo Compilation successful!
  )

This batch file:
- Uses paths from rb.config
- Includes SDL2 headers and libraries
- Creates program.exe (hardcoded)
- Shows error or success message


WHY PROGRAM.EXE IS HARDCODED:
------------------------------

Design decision in RB BASIC:
- Simplifies the compilation process
- output.c is always the intermediate file
- program.exe is always the output
- User can rename program.exe after compilation if needed
- Matches QuickBASIC behavior where compiled programs
  were often just "program.exe" during development

TROUBLESHOOTING:
----------------

"Failed to open file" on command line:
  - Check quotes are being stripped (main.cpp fix)
  - Verify path exists
  - Try without spaces in path first

"Transpilation FAILED":
  - Check rbbasic.exe exists in parent folder
  - Check .bas file has valid BASIC syntax
  - Look at error messages in output panel
  - Try running rbbasic.exe manually:
      cd to your .bas directory
      ..\rbbasic.exe yourfile.bas

"compile.bat not found":
  - rbbasic.exe didn't complete successfully
  - Check transpilation errors
  - Verify rbbasic.exe can write files

"Compilation FAILED" (step 2):
  - compile.bat ran but GCC failed
  - Check rb.config has correct MinGW path
  - Check output.c for syntax errors
  - Try running compile.bat manually:
      cd to your .bas directory
      compile.bat

"program.exe not found" when running:
  - Compilation didn't succeed
  - Check if program.exe exists in source directory
  - Compile first before running

"program.exe runs but shows nothing":
  - Program might have crashed
  - Check if it needs SDL2.dll
  - Try running program.exe manually from Explorer


COMPARISON TO QB64:
-------------------

QB64 Workflow:
  - Single executable
  - Hides intermediate C code
  - Custom C++ backend
  - Automatic library linking

RB BASIC Workflow:
  - Separated transpiler and compiler
  - Shows intermediate files (educational)
  - Uses standard GCC
  - Manual library configuration (rb.config)
  - More transparent process

Both create temporary intermediate files, but:
- QB64 hides them in internal folder
- RB BASIC creates them alongside your .bas file
